Introduction¶

Quote from Kruschke2013 (p. 583), paraphrased and adapted to the situation:

Suppose I think that a realistic sample size for my study will have $N = 50$. Suppose I have several goals for the analysis, selected here more for illustrative purposes than for realism. With regard to the magnitude of $\Delta\alpha$ (the difference between the doctors and the softwares output for any image), suppose I would like to show that its 95% HDI (highest density posterior interval; a.k.a. HPD) falls entirely within a ROPE (range of practical equivalence) of $(-5^\circ, 5^\circ)$, which is to say that after these experiments we believe the difference between the doctors and the softwares prediction is not more different than $5^\circ$.

We can increase power by increasing the experiments sample size or by targeting a simpler goal (e.g. a wider ROPE).

Figure 6 of Kruschke2013:

Flowchart for estimating Bayesian power. At the left, a hypothetical distribution of parameter values is used to repeatedly generate representative credible parameter values. For each set of parameter values, a simulated sample of data is generated from the model. Then Bayes rule is applied to derive the posterior distribution from the simulated data and assay whether the various goals have been achieved for that sample. Across many replications of simulated experiments, the probability of achieving each goal is estimated to arbitrarily high accuracy.

Procedure¶

  1. Take all the labeled images we have.
  2. Fit the statistical model to the training data to define the plausible parameter values.
  3. Using this posterior, simulate $S$ batches of $N$ samples. (Note: steps 13 of inferring a distribution of plausible parameter values can be done with a Bayesian model even if we intend to provide confidence, and they would result in better estimates for power analysis than a simple point estimate of the parameters.)
  4. Fit the model independently to these $S$ batches of generated data.
  5. Count the number of batches where the 95% HDI fits entirely within the ROPE ($n_{succ}$), and where it falls entirely outside the ROPE ($n_{fail}$).
  6. Assuming a uniform prior on a correct software (with differences from ground truth inside the ROPE), we can use the binomial observations $n_{succ}$ out of $S$ to update this model, resulting in a $\text{Beta}(1+n_{succ}, 1 + S - n_{succ})$ posterior for demonstrably correct software. (Using $n_{fail}$, we can do a similar analysis for demonstrably incorrect software.) (Note: we can use the binomial model in this step even if we do the previous steps using confidence intervals or any other success criterion.)

The statistical model¶

Instead of directly modeling the angles returned by the doctors and the software, I first model the distribution of the three points A, B, C, then the output of both the doctors ($A_{doc}$) and the software ($A_{sw}$) as the true point some error ($\sigma_{doc}$ and $\sigma_{sw}$), and from these I calculate the angles and the error of the software ($\Delta\alpha = \alpha_{doc} - \alpha_{sw}$). (It would make perfect sense to model $\Delta\alpha$ as $\alpha_{true} - \alpha_{sw}$ instead of $\alpha_{doc} - \alpha_{sw}$, but that would take some arguing.) This allows me to estimate the angle error much more precisely, instead of just blindly assuming e.g. a normal distribution on $\Delta\alpha$.

Note that even the doctors dont know the true point, but this need not adversely affect the calculations: in step 4 (and during the clinical trial), we can simply assume a near-zero value of $\sigma_{doc}$.

Modeling the 3 points throughout the whole population as independent distributions would be a bad fit they are obviously not independent (e.g. hip size affects all). It is better to use a coordinate system with one of the 3 points of the expert evaluation as the origin, instead of using a global coordinate system where all 3 points vary. I chose the inner point of the hip as the origin so that the angle is always positive therefore $C_{true} = C_{doc} = C_{sw} = (0,0)$. (In practice this assumption would need some fine-tuning, because this now assumes that one point is always estimated precisely).

Points where Bayesian estimation shines¶

Hierarchical models¶

Hierarchical models can be handled easily in a Bayesian setting, whereas they are considerably more complicated using frequentist statistics. Such models could be used for many purposes, and they all improve the precision of the model.

a) Model the uncertainty in the points, instead of the angles.

b) Combine multiple expert evaluations of the same image in the same model, thus decreasing the models uncertainty in the expert opinions.

c) By keeping track of which doctor made the analysis, we can assign different carelessness factors to the each doctor, instead of using the same $\sigma_{doc}$ for all of them.

Power analysis¶

According to Kruschke2013, Bayesian power analysis results in more precise power estimates than NHST (null hypothesis significance testing) power analysis:

In either retrospective or prospective power analyses, NHST uses a point value for the hypothetical effect size. In Bayesian power analysis, one uses an entire distribution of parameters instead of a single point value for the effect size. Thus, every value of effect size is considered but only to the extent that it is considered to be credible. NHST power analysis can consider various point values, such as the end points of a confidence interval, but the different point values are not weighted by credibility and therefore can yield a huge range of powers. As a consequence, NHST power analysis often yields extremely uncertain results (e.g., Gerard et al., 1998; Miller, 2009; Thomas, 1997), but Bayesian power analysis yields precise estimates of power.

Adaptive experiment design¶

Stopping an experiment early (when the results are conclusive) doesnt require further care in a Bayesian setting, whereas multiple books and papers are written on adaptive design for frequentist statistics, as the testing intentions affect the confidence intervals.

def truncated_multivariate_normal(mean, sigma):
    """Returns a sample from a truncated multivariate normal distribution
    
    truncated to 2 sigma around the mean."""
    assert isinstance(sigma, float) or isinstance(sigma, int)
    proposal = np.array(np.inf)
    while np.linalg.norm(proposal - mean) > 2 * sigma:
        proposal = st.multivariate_normal(mean=mean, cov=np.ones_like(mean) * sigma).rvs(1)
    return proposal

Learn the data generating distribution¶

We use the Python interface of the statistical modeling software Stan. Instead of modeling the end result (the angles, or the difference between the angle predicted by the doctor and the software), we model the variance around the true points, both for the doctors and the software.

Stan has its own modeling language, and requires compiling the model before sampling.

In [381]:
model_code = """

functions {
    /* Given 2-dimensional vectors A and B,
       calculate the angle ABO */
    real angle(vector A_pt, vector B_pt)
    {
        vector[2] v1;
        vector[2] v2;
        
        v1 = A_pt - B_pt;
        v2 = -B_pt;
        
        return acos(dot_product(v1, v2) /
                    sqrt(dot_product(v1, v1) * dot_product(v2, v2))) \
                    * 180 / pi();
    }
}

data {
    int<lower=0> N; // number of data points
    vector[2] A_doc[N];
    vector[2] B_doc[N];
    vector[2] A_sw[N];
    vector[2] B_sw[N];
    
    real<lower=0> sigma_doc; // some small number
}

parameters {
    // log(sigma) has an improper uniform prior on it,
    //  for A_population_sigma, B_population_sigma, sigma_sw
    real log_sigma_sw;

    real log_A_population_sigma;
    real log_B_population_sigma;

    // A_population_mean and B_population_mean also have an improper uniform prior on them
    vector[2] A_population_mean;
    vector[2] B_population_mean;
    
    vector[2] A_true[N];
    vector[2] B_true[N];
    
    // These are points sampled from the posterior,
    //  not the observed points.
    vector[2] A_true_;
    vector[2] B_true_;
    vector[2] A_doc_;
    vector[2] A_sw_;
    vector[2] B_doc_;
    vector[2] B_sw_;
}

transformed parameters {
    real<lower=0> A_population_sigma;
    real<lower=0> B_population_sigma;
    real<lower=0> sigma_sw;
    
    real alpha_doc;
    real alpha_sw;   
    real alpha_diff;
    
    A_population_sigma = exp(log_A_population_sigma);
    B_population_sigma = exp(log_B_population_sigma);
    sigma_sw = exp(log_sigma_sw);
    
    alpha_doc = angle(A_doc_, B_doc_);
    alpha_sw = angle(A_sw_, B_sw_);

    alpha_diff = alpha_doc - alpha_sw;
}

model {
    for (n in 1:N) {
        A_true[n] ~ normal(A_population_mean, A_population_sigma);
        B_true[n] ~ normal(B_population_mean, B_population_sigma);

        A_doc[n] ~ normal(A_true[n], sigma_doc);
        B_doc[n] ~ normal(B_true[n], sigma_doc);
        
        A_sw[n] ~ normal(A_true[n], sigma_sw);
        B_sw[n] ~ normal(B_true[n], sigma_sw);
    }


    A_true_ ~ normal(A_population_mean, A_population_sigma);
    B_true_ ~ normal(B_population_mean, B_population_sigma);
    
    A_doc_ ~ normal(A_true_, sigma_doc);
    B_doc_ ~ normal(B_true_, sigma_doc);

    A_sw_ ~ normal(A_true_, sigma_sw);
    B_sw_ ~ normal(B_true_, sigma_sw);
}
"""

model = pystan.StanModel(model_code=model_code)

Aside: what is the meaning of 95% credible interval on alpha_diff?¶

It means that if we begin the experiment with assigning credibilities to parameter values as the prior describes, and the model is true, then after the experiment we should be 95% certain that the parameter value is in the 95% credible interval (e.g. that the output difference on the same image between doctor and software is in the range of $[-7.4, 7.9]$, if the population is as described by the model).

Aside: what is the meaning of 95% confidence interval on $\overline{\Delta\alpha}$, given an experiment?¶

A 95% confidence interval $[\Delta\alpha_1, \Delta\alpha_2]$ means that if the true value of $\overline{\Delta\alpha}$ is outside this interval, then 5% of the experiments will show a more extreme test statistic than what was observed in this experiment.

Assuming a t-test is performed, the test statistic would be the t-statistic, and the meaning of CI would be: if we assume $\Delta\alpha$ to be normally distributed with a given mean, then for any value $\delta$ inside the CI the observations are compatible with the hypothesis that the mean is $\delta$, whereas $\delta$ values outside the CI are quite incompatible. Note that this doesnt say anything about how wide is the distribution of $\Delta\alpha$ around $\delta$, as is demonstrated by the example below.

The 95% CI of (-0.92, 0.67) means that the average error is unlikely to be 2.3, but who cares? The errors vary in the range of $\pm5^\circ$!

Furthermore, we can make the CI arbitrarily small, simply by using more samples. We are not changing the distribution of $\Delta\alpha$ at all, we are only increasing the precision of our estimate of the average error of the software

Determine the power of the experiment¶

We start out with a uniform prior about what we believe the power of the experiment could be.

In [371]:
power_probabilities = np.linspace(0,1, 100)   # parameter space for our power estimation
power_prior = st.uniform.pdf(power_probabilities)

And update the prior according to the Binomial(S, p) likelihood, where S is the number of experiments simulated and p is the power.
